# Support

Doctor link is community-supported software. There is no guaranteed response time or production SLA.

## Before opening an Issue

1. Upgrade to the latest published version.
2. Run `doctor-link --version` and `doctor-link preflight --json <project>`.
3. Remove tokens, credentials, personal data, private source, and confidential logs.
4. Reduce the problem to the smallest reproducible example when possible.

Use the bug template for incorrect behavior and the feature template for a concrete product improvement. General usage questions may use GitHub Discussions when enabled; otherwise open a sanitized Issue and select the closest template.

For vulnerabilities or sensitive privacy failures, follow [SECURITY.md](SECURITY.md) and do not file the details publicly.
