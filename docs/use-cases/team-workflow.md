# Use Case: Team Workflow

Doctor link helps teams share diagnostic context across users, developers, reviewers, and AI Coding tools.

## Problem

Teams often lose debugging context across chat threads, issue comments, local logs, and PRs.

## Doctor link workflow

1. Keep diagnostic packages under `DoctorReports/`.
2. Use before / after workflow for changes.
3. Upload diagnostic artifacts in CI.
4. Review project health summaries.
5. Use final audits before closing phases.

## Value

- Shared evidence package.
- Repeatable CI diagnostics.
- Clear audit trail.
- Lower risk of closing issues without verification.
