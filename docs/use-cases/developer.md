# Use Case: Developer

Doctor link helps developers turn vague bug reports into reproducible diagnostic packages.

## Problem

Developers need evidence, reproduction steps, and verification criteria before changing code.

## Doctor link workflow

1. Scan or collect project evidence.
2. Record test results and reproduction outputs.
3. Compare before and after reports.
4. Run verification.
5. Export a reviewed diagnostic package when needed.

## Value

- Faster triage.
- Clearer reproduction.
- Safer handoff to AI or other developers.
- Verification-driven closure.
