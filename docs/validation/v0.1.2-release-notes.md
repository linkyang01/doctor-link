# Doctor link v0.1.2

Release date: 2026-07-13

Doctor link `v0.1.2` repairs the end-to-end diagnostic workflow, strengthens local command and privacy boundaries, adds release-grade package validation, proves cross-platform operation in GitHub Actions, and expands repository documentation.

## Highlights

- Restored working collect, verify, compare, strategy, handoff, workbench, assertion, and risk-review flows.
- Added read-only `doctor-link preflight` readiness checks.
- Replaced shell execution paths with a safe argument-vector command runner.
- Added AI handoff profiles for Grok Build, Windsurf, and Cline.
- Added Python 3.10–3.12 and Ubuntu/macOS/Windows GitHub validation.
- Added security, dependency, coverage, distribution-content, Twine, and isolated-install gates.
- Published detailed README, contribution, security, GitHub workflow, validation, and release documentation.

## User-facing changes

### Preflight before diagnosis

```bash
doctor-link preflight . --json
```

Preflight checks Python readiness, Doctor link configuration, strategy, configured commands, reproduction/test catalogs, and local tools without executing repository commands.

### Evidence and verification repairs

The CLI and runtime interfaces for evidence collection, user assertions, verification write-back, report comparison, workbench JSON output, AI result/history, and risk review are aligned again. Compatibility aliases preserve documented command forms such as `--logs` and `--attachment`.

### AI handoff expansion

```bash
doctor-link handoff list
doctor-link handoff check <package_dir> --tool cursor --json
doctor-link handoff <package_dir> --tool codex --json
doctor-link wizard --folder . --tool grok --handoff --json
```

Supported profiles include Codex, Cursor, Cline, Windsurf, Grok Build, and generic handoff workflows.

### Safer command execution

Reproduction and test-matrix commands no longer run through a shell. Normal arguments and reviewed sequential `&&` commands are supported, while pipelines, redirection, command substitution, and unsafe shell operators are rejected.

Adapter and plugin execution remains dry-run by default and requires explicit `--allow-run` approval.

## Packaging and installation

Download the wheel or source archive from this GitHub Release, then install the wheel:

```bash
python -m pip install doctor_link-0.1.2-py3-none-any.whl
doctor-link --version
doctor-link preflight . --json
```

Install from source:

```bash
git clone https://github.com/linkyang01/doctor-link.git
cd doctor-link
git checkout v0.1.2
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e .
doctor-link --version
```

Windows PowerShell activation:

```powershell
.\.venv\Scripts\Activate.ps1
```

PyPI publication is not part of this GitHub Release unless separately authorized and completed.

## Security and privacy

- Diagnostic data stays local unless a user intentionally shares it.
- Review `.doctorlink/` commands, adapter manifests, and plugin manifests before execution.
- Review redaction and privacy output before sharing diagnostic packages or CI artifacts.
- Do not attach secrets, customer data, private source, or unredacted diagnostic packages to public issues.

See `SECURITY.md`, `docs/sensitive-data-handling.md`, and `docs/export-security-checklist.md`.

## Compatibility

- Supported Python: 3.10, 3.11, 3.12.
- Validated operating systems: GitHub-hosted Ubuntu, macOS, and Windows.
- Package format remains a pure-Python wheel plus source distribution.
- Existing local-first diagnostic package workflows remain supported.

## Known boundaries

This release does not provide:

- a hosted web platform or cloud synchronization;
- user accounts, telemetry, or a marketplace;
- production signing-key infrastructure;
- hosted enterprise archive or diagnostic knowledge base;
- production RBAC;
- customer-environment or production acceptance certification.

## Upgrade notes

Run preflight after upgrading:

```bash
doctor-link --version
doctor-link preflight . --json
```

Review any reported configuration warnings before running repository commands. Teams using adapters or plugins should revalidate manifests and continue to require explicit `--allow-run` approval.
