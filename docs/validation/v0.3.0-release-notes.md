# Doctor link v0.3.0

Doctor link `v0.3.0` extends verified automatic problem solving from Python to Node.js JavaScript and TypeScript projects without weakening the existing repair-safety contract.

## Highlights

- Detects Node.js projects from `package.json` and common JavaScript/TypeScript source roots.
- Discovers `npm test`, `pnpm test`, `yarn test`, or `bun run test` from package metadata and lockfiles.
- Falls back to `node --test` for compatible JavaScript test files when no usable package test script exists.
- Protects package manifests, lockfiles, workspace files, tests, and common JavaScript/TypeScript test-runner configuration during repair.
- Blocks missing toolchains and timed-out required baseline checks before creating a branch or invoking Codex.

## Verified repair contract

Automatic repair still requires a clean Git repository, a reproducible failing check, and explicit `--allow-repair`. Doctor link creates an isolated repair branch, limits Codex to three rounds, preserves local receipts, and independently reruns the original commands. A modified protected input cannot produce ordinary `verified`; explicitly authorized verification changes return `review_required`.

## Installation

After GitHub Release publication, install the attached wheel:

```bash
python -m pip install ./doctor_link-0.3.0-py3-none-any.whl
doctor-link --version
```

Doctor link remains a Python 3.10+ CLI. Node.js and the package manager selected by the target project are required only for JavaScript/TypeScript automatic repair. PyPI publication is not implied by this GitHub Release and occurs only when separately authorized.
