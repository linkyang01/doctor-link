# v0.5.0 Release Notes (Draft)

Version 0.5.0 introduces the shortest Doctor link workflow yet:

```bash
doctor-link assist /path/to/project --problem "Checkout duplicates a charge"
```

Doctor link now ranks existing tests and project test entrypoints against the reported problem, executes safe candidates without test-cache pollution, and selects a command only when it observes a real failure. The guided result is written as JSON and a local browser page before the existing explicit repair gate.

The new corpus materializes 24 isolated Git projects across Python and Node.js. It covers payments, authorization, cache consistency, Unicode, concurrency, configuration, retry accounting, pagination, timezone conversion, transaction rollback, path traversal, and rate limits. The current corpus run reproduced 24/24 problems with 100% expectation matching and no repair invocation.

Local validation currently passes 363 tests at 85.06% branch-aware coverage, Ruff, Bandit with zero medium/high findings, and dependency auditing with no known vulnerabilities. The wheel and source archive pass distribution-content and Twine checks. The installed-wheel full-capability lab passes 66/66 routes through 75 invocations and twelve scenario checks.
