# v0.5.0 Description-to-reproduction

Version 0.5.0 closes the largest usability gap between a user-observed problem and Doctor link's verified repair loop.

## Outcomes

1. Natural-language problem descriptions are tokenized, expanded through bounded domain aliases, matched to existing test files and project-declared test entrypoints, and optionally executed as safe candidate checks.
2. `doctor-link assist` asks only for a project and problem, produces a local browser result, and stops at the existing explicit repair approval gate unless `--allow-repair` is supplied.
3. A 24-project corpus covers twelve fault domains in Python and Node.js and proves that every ordinary-language description becomes a genuine failed-check reproduction.

## Safety contract

- candidate commands come only from existing test files or project metadata;
- shell control operators remain rejected by the existing safe runner;
- irrelevant test filenames are not claimed as focused matches;
- a candidate that changes the Git working tree is never trusted;
- generated results escape user-controlled HTML;
- repair still requires a clean Git repository, explicit approval, an isolated branch, protected inputs, and independent verification.

## Release gate

The full source suite, 24-project corpus, 66-route installed-wheel lab, build/content/Twine checks, security checks, and the complete GitHub Actions matrix must pass before publication.
