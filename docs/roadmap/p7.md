# Doctor link P7 Roadmap

P7 converts selected Doctor link planning, specification, documentation-only, and deferred capabilities into real runtime/product capabilities.

P7 must not treat documents alone as implementation completion.

## Current baseline

Doctor link has completed:

- P0-P5.10 mainline CLI and productization work;
- GitHub Actions cloud validation;
- Doctor link self-validation;
- P6.1-P6.11 planning, schema, specification, conformance, ecosystem, and closure documentation.

Known remaining non-runtime or deferred areas include:

- local / target-environment validation execution;
- P2+ deferred workbench UX and write-back items;
- P3 real AI tool runtime handoff helpers;
- P4 stronger operational automation;
- P5 release execution and distribution actions;
- P6 runtime implementations for adapter, plugin, integrity, privacy, knowledge base, and enterprise archive.

## P7 completion rule

A P7 runtime item is complete only when it includes:

- implementation code;
- CLI or runtime behavior where applicable;
- schemas or configuration validation where applicable;
- tests and fixtures;
- documentation;
- CI validation;
- acceptance evidence.
