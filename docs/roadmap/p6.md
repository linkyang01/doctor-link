# Doctor link P6 Roadmap

P6 turns Doctor link from a productized diagnostic tool into a diagnostic protocol standard and ecosystem platform for AI Coding workflows.

P6 has completed P6.1 schema standardization, P6.2 local compatibility / conformance validation, P6.3 Adapter SDK planning, P6.4 Plugin SDK planning, P6.5 third-party AI Coding integration specification, P6.6 signing and integrity specification, and P6.7 privacy and security level specification. P6.8 and later remain future work unless separately authorized.

Current boundary:

- P6.1 Diagnostic Package Schema v1: completed.
- P6.2 Compatibility and conformance test suite: completed.
- P6.3 Adapter SDK planning: completed as planning specification only.
- P6.4 Plugin SDK planning: completed as planning specification only.
- P6.5 Third-party AI Coding integration specification: completed as planning specification only.
- P6.6 Diagnostic package signing and integrity: completed as planning specification only.
- P6.7 Privacy and security level specification: completed as planning specification only.
- P6.8+ governance, knowledge base, and public ecosystem work: not started.
- No real permission system, cloud service, account system, live third-party integration, real signing implementation, key management, Adapter SDK runtime, Plugin SDK runtime, Web platform, marketplace, PyPI publishing, release tag, or GitHub Release is authorized by P6.7.
