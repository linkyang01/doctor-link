## CLI entrypoint review

The package entrypoint remains:

```toml
[project.scripts]
doctor-link = "doctor_link.p4_cli:main"
```

`doctor_link.p4_cli` imports the base Click group from `doctor_link.cli` and registers the P4 command groups on the same CLI surface. This keeps earlier commands such as `scan`, `report`, `collect`, `assert`, `verify`, `view`, and `handoff` available while adding P4 commands such as `reproduce`, `test`, `diagnose`, and `health`.

This structure is acceptable for P5.9 because it preserves backward-compatible command names and avoids changing the public CLI contract during release hardening.
