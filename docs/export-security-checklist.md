# Export Security Checklist

Use this checklist before sharing a Doctor link diagnostic package or exported zip.

## Package review

- [ ] Review `summary.md`.
- [ ] Review `problem-map.md`.
- [ ] Review `timeline.md`.
- [ ] Review `evidence-list.md`.
- [ ] Review `doctor-report.json`.
- [ ] Review `ai-task.md`.
- [ ] Review `user-assertions.json`.
- [ ] Review `verification-result.json`.

## Evidence review

- [ ] Review `evidence/logs/`.
- [ ] Review `evidence/commands/`.
- [ ] Review `evidence/probes/`.
- [ ] Review `evidence/attachments/`.
- [ ] Review `evidence/reproductions/`.
- [ ] Review `evidence/test-results/`.

## Sensitive data checks

- [ ] No API keys.
- [ ] No passwords.
- [ ] No private tokens.
- [ ] No unnecessary customer data.
- [ ] No private screenshots unless required.
- [ ] No unnecessary attachments.

## Sharing decision

- [ ] Package contains only evidence needed to diagnose or verify the issue.
- [ ] Package has been reviewed by a human.
- [ ] Sharing destination is approved.
